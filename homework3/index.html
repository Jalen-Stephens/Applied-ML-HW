Problem 1. Reading a graph as a matrix

Path graph: \(1 - 2 - 3 - 4\), features \(x = (1,0,2,1)^\top\).

(a) Adjacency matrix

\[A = \begin{bmatrix}0&1&0&0\\1&0&1&0\\0&1&0&1\\0&0&1&0\end{bmatrix}.\]

(b) Compute \(Ax\)

\[(Ax)_1 = 0(1)+1(0)+0(2)+0(1)=0,\] \[(Ax)_2 = 1(1)+0(0)+1(2)+0(1)=3,\] \[(Ax)_3 = 0(1)+1(0)+0(2)+1(1)=1,\] \[(Ax)_4 = 0(1)+0(0)+1(2)+0(1)=2.\] \[Ax = \begin{bmatrix}0\\3\\1\\2\end{bmatrix}.\]

(c) Interpretation

\((Ax)_i\) is the sum of features of neighbors of node \(i\) that lie one hop away. This is the basic neighborhood aggregation that Lecture 15 describes as the core GNN operation.

(d) \(A^2 x\)

Compute \(A(Ax)\) with \(Ax=(0,3,1,2)^\top\):

\[(A^2x)_1=0(0)+1(3)+0(1)+0(2)=3,\] \[(A^2x)_2=1(0)+0(3)+1(1)+0(2)=1,\] \[(A^2x)_3=0(0)+1(3)+0(1)+1(2)=5,\] \[(A^2x)_4=0(0)+0(3)+1(1)+0(2)=1.\] \[A^2x = \begin{bmatrix}3\\1\\5\\1\end{bmatrix}.\]

\(A^2\) counts length-2 walks, so \((A^2x)_i\) aggregates features from nodes reachable in exactly two steps. Stacking GNN layers deepens the receptive field in the same way.

Problem 2. One round of message passing

Same graph, \(h^{(0)}=(1,0,2,1)^\top\). Update: \(m_v^{(1)}=\sum_{u\in\mathcal N(v)}h_u^{(0)}\), \(h_v^{(1)}=h_v^{(0)}+m_v^{(1)}\).

(a) Messages

\[m_1^{(1)}=h_2^{(0)}=0,\quad m_2^{(1)}=h_1^{(0)}+h_3^{(0)}=1+2=3,\] \[m_3^{(1)}=h_2^{(0)}+h_4^{(0)}=0+1=1,\quad m_4^{(1)}=h_3^{(0)}=2.\]

(b) Updated representations

\[h_1^{(1)}=1+0=1,\ h_2^{(1)}=0+3=3,\ h_3^{(1)}=2+1=3,\ h_4^{(1)}=1+2=3.\]

(c) Result

\[h^{(1)}=(1,3,3,3)^\top.\]

(d) Why 1-hop only

Each \(m_v^{(1)}\) sums only the features of direct neighbors \(\mathcal N(v)\). No neighbor-of-neighbor information enters the sum, so the representation after one round encodes only the neighborhood within one hop.

Problem 3. Two rounds of message passing

Continue with \(h^{(1)}=(1,3,3,3)^\top\).

(a) Compute

\[m_1^{(2)}=3,\quad m_2^{(2)}=1+3=4,\quad m_3^{(2)}=3+3=6,\quad m_4^{(2)}=3.\] \[h^{(2)}=(1+3,\;3+4,\;3+6,\;3+3)^\top=(4,7,9,6)^\top.\]

(b) Influence on node 2

All four nodes eventually influence the representation at node 2. For instance, node 4 reaches node 2 along the path \(4\to3\to2\) in two hops.

(c) Why \(t\)-hop locality

The claim follows by induction. After round 1 each node knows its neighbors one hop away. In round 2 each node aggregates those round-1 representations, which extends the horizon by one hop. After \(t\) rounds, the receptive field covers exactly the neighborhood within \(t\) hops.

Problem 4. ReLU version

Graph \(1-2-3\), \(h^{(0)}=(-1,2,-2)^\top\), update: \(h_v^{(1)}=\operatorname{ReLU}(h_v^{(0)}+m_v^{(1)})\).

(a) Messages

\[m_1^{(1)}=2,\quad m_2^{(1)}=-1+(-2)=-3,\quad m_3^{(1)}=2.\]

(b) Update

Pre-activations: \((1,\;-1,\;0)\). After ReLU:

\[h^{(1)}=(1,0,0)^\top.\]

(c) Effect of nonlinearity

ReLU clips negative aggregated values to zero, introducing nonlinearity and sparsity. Without it, stacking layers is just repeated linear aggregation, which is equivalent to multiplying by powers of \(A\). The nonlinearity is what makes deep GNNs more expressive than a single matrix power.

Problem 6. 1-WL on a cycle graph

Cycle \(C_4\): \(1-2-3-4-1\), all start with color \(a\).

(a) Two rounds

Round 1: every node has degree 2 and signature \((a,\{a,a\})\) → all get color \(b\).
Round 2: every node has signature \((b,\{b,b\})\) → all get color \(c\). Same single-class partition.

(b) What happens

Colors rename but the partition into one class never changes. The process stabilizes after round 1.

(c) Why 1-WL cannot distinguish nodes

\(C_4\) is vertex-transitive: every node has the exact same local structure at every depth. Since 1-WL only looks at the current color together with the multiset of neighbor colors, and this pair is identical for all nodes, no refinement occurs.

Problem 8. \(C_6\) vs two triangles

(a) Degrees

Every node has degree 2 in both \(G_3=C_6\) and \(G_4=K_3\sqcup K_3\).

(b)-(c) Two WL rounds

Round 1: all nodes get signature \((a,\{a,a\})\) → one color. Round 2: \((b,\{b,b\})\) → still one color. Both graphs: single class of size 6.

(d) Distinguished?

No. 1-WL produces identical color multisets for both graphs.

(e) Warning for MPNNs

Standard sum or mean aggregation MPNNs are at most as expressive as 1-WL according to the Jegelka survey. If 1-WL fails, the MPNN fails too. The fact that \(C_6\) is connected while \(K_3\sqcup K_3\) is not is a fundamental structural difference, yet local message passing cannot detect it.

Problem 9. Manual GNN vs 1-WL

Update: \(h_v^{(t)}=h_v^{(t-1)}+\sum_{u\in\mathcal N(v)}h_u^{(t-1)}\), all \(h_v^{(0)}=1\).

(a) \(h^{(1)}\)

Every node has degree 2: \(h_v^{(1)}=1+1+1=3\) for all \(v\), in both \(G_3\) and \(G_4\).

(b) \(h^{(2)}\)

\(h_v^{(2)}=3+3+3=9\) for all \(v\), in both graphs.

(c) Observation

The GNN yields identical representations in both graphs at every round, so it cannot distinguish them, just as 1-WL cannot.

(d) Intuitive connection

Both 1-WL and sum-aggregation GNNs update each node by combining its state with a permutation-invariant summary of neighbors. When all local neighborhoods are identical, for example in regular graphs, both procedures produce identical outputs. This is the "GNN ≤ 1-WL" expressiveness result.

Problem 10. Permutation invariance

(a)

Relabeling permutes node indices. The adjacency matrix changes from \(A\) to \(PAP^\top\) but the underlying graph is identical.

(b)

A graph's class depends on structure, not on arbitrary node numbering. The model must give the same output regardless of how nodes are indexed.

(c) Invariant readout

\(\text{readout}=\sum_v h_v^{(T)}\). Summation is commutative, so it is permutation invariant. Mean and max pooling also work.

(d) Non-invariant readout

Output only \(h_1^{(T)}\) depends on which node is labeled "1."

Problem 11. Sum aggregation

(a)

A multiset is like a set but allows repeated elements: \(\{a,a,b\}\neq\{a,b\}\).

(b)

Multiple neighbors can share the same feature, so neighborhoods are naturally multisets.

(c)

Sum is order-invariant and fixed-size regardless of degree. Concatenation depends on ordering and degree.

(d)

\(\{1,1,1\}\) and \(\{1\}\) both have mean 1 but sums 3 vs 1. Sum preserves degree information that mean normalizes away.

Problem 13. Limits of message passing

(a)

Diameter is the longest shortest path between any two nodes, so it is an inherently global quantity. A \(t\)-hop receptive field cannot see beyond radius \(t\), so short message passing cannot encode the full diameter.

(b)

Counting long cycles requires seeing how the cycle closes globally, but 1-WL-equivalent MPNNs cannot distinguish regular graphs that differ only in cycle structure.

(c)

A local property is something like node degree, which you read off a neighborhood. A global property is something like whether the graph is connected, which depends on the entire structure.

(d)

A shallow GNN handles degree-based tasks easily but will likely fail on tasks that require true connectivity or diameter information.

Problem 14. Higher-order ideas

(a)

Node pairs can directly encode relational structure such as shared neighbors or specific paths, while single-node representations must infer those relations only indirectly.

(b)

The number of states scales as \(O(n^k)\) when you color \(k\)-tuples of nodes, which becomes impractical for large graphs.

(c)

There is a direct tension: more expressive models cost more. Practical research therefore looks for middle-ground approaches, including subgraph GNNs and local higher-order methods.

Problem 15. Build the Laplacian

(a) \(A\)

Path graph \(1-2-3-4\):

\[A=\begin{bmatrix} 0 & 1 & 0 & 0\\ 1 & 0 & 1 & 0\\ 0 & 1 & 0 & 1\\ 0 & 0 & 1 & 0 \end{bmatrix}.\]

(b) \(D\)

\[D=\operatorname{diag}(1,2,2,1).\]

(c) \(L=D-A\)

\[L=\begin{bmatrix}1&-1&0&0\\-1&2&-1&0\\0&-1&2&-1\\0&0&-1&1\end{bmatrix}.\]

(d) Verify \(L\mathbf 1=0\)

\[L\mathbf 1=\begin{bmatrix}1-1\\-1+2-1\\-1+2-1\\-1+1\end{bmatrix}=\begin{bmatrix}0\\0\\0\\0\end{bmatrix}.\]

The constant vector is an eigenvector with eigenvalue 0: if all nodes have the same value, there is no local disagreement.

Problem 16. Laplacian as difference operator

\(f=(1,3,3,1)^\top\).

(a) Compute \(Lf\)

\[(Lf)_1=1(1)-1(3)=-2,\quad (Lf)_2=-1(1)+2(3)-1(3)=2,\] \[(Lf)_3=-1(3)+2(3)-1(1)=2,\quad (Lf)_4=-1(3)+1(1)=-2.\] \[Lf=(-2,2,2,-2)^\top.\]

(b) Interpretation

\((Lf)_i=\sum_{j\sim i}(f_i - f_j)\) measures how much node \(i\) exceeds its neighbors. Nodes 2 and 3 are local peaks with value \(+2\); nodes 1 and 4 are local valleys with value \(-2\).

(c) Flat vs different

No node is perfectly flat. Having \(Lf=0\) would mean each node matches its neighbors. Nodes 2 and 3 behave like peaks and nodes 1 and 4 like valleys.

Problem 17. Adjacency vs Laplacian mixing

(a) \(Af\)

\[Af=(3,4,4,3)^\top.\]

(b) Comparison

\(Lf=Df-Af=(1,6,6,1)^\top-(3,4,4,3)^\top=(-2,2,2,-2)^\top\). Checks out.

(c) Difference

\(A\) is a mixing operator: it replaces each node's value with the sum of its neighbors. \(L\) is a contrast operator: it measures how much each node differs from its neighbors. In GNN terms, \(A\) corresponds to the aggregation step, while \(L\) relates to measuring local smoothness from a spectral viewpoint.

Problem 18. A first spectral filter

\(g=(I-\alpha L)f\), \(\alpha=\tfrac12\).

(a) Compute

\[\tfrac12 Lf=(-1,1,1,-1)^\top,\] \[g=f-\tfrac12 Lf=(1,3,3,1)^\top-(-1,1,1,-1)^\top=(2,2,2,2)^\top.\]

(b) More or less similar?

More similar. The original \(f\) ranges from 1 to 3; after filtering, \(g\) is perfectly constant at 2. The filter smoothed the signal completely.

(c) Why useful

Graph signals often exhibit homophily, meaning connected nodes tend to have similar features or labels. Smoothing reduces noise and encourages similar representations for neighboring nodes. That behavior acts like graph-structure-aware regularization.

(d) Connection to message passing

\(g=(I-\alpha L)f = (1-\alpha D)f + \alpha Af\): a weighted mix of a node's own feature and the sum of its neighbors' features. This is exactly one linear message-passing layer. Spectral filtering with a first-order polynomial in \(L\) is mathematically equivalent to one GNN round.

Problem 20. Population vs empirical risk

(a)

Empirical risk is the average loss on training graphs. Population risk is the expected loss over the true distribution of graphs. The gap between them is the generalization error.

(b)

Overfitting means the model memorizes patterns that are specific to the training graphs instead of learning rules that generalize.

(c)

Graphs vary in size, degree, and motifs. A model trained on small graphs may therefore face entirely different structural regimes at test time.

Problem 21. Extrapolation and graph size

(a)

Larger graphs introduce longer-range dependencies and structural patterns absent from training data. Statistics of learned features can shift.

(b)

If local neighborhoods look similar across graph sizes, the GNN can transfer what it learned, because locality is the key inductive bias.

(c)

The model may have learned shortcuts that only work at training scale, such as implicitly assuming small graphs are always connected, and those shortcuts break on larger graphs.

Problem 22. Algorithmic alignment

(a)

An architecture is "aligned" with a task when its computation mirrors the steps of an algorithm that solves the task.

(b)

Bellman-Ford iteratively relaxes distances via neighbor updates, which is exactly what message passing implements at each step.

(c)

Label propagation and diffusion processes on graphs are naturally expressed as iterative neighbor aggregation.

Part VI. Reflection (10 pts)

The realization that both 1-WL and my toy GNN would fail to distinguish the graphs and the union of two triangles came as a surprise. In each case, there is a very different global topology, path vs. two disjoint edges, but all nodes look identical at the local level. Each node has degree 2, and their neighbors have degree 2 too. This really helped me see just how severely local aggregation is limited. It reinforced for me why people study higher-order architectures.

I also appreciate the Laplacian portion of the course. Each matrix is an example of a way you can combine the neighborhood information of a message-passing algorithm like GNNs, whereas each measure of local disagreement is similar to how a sharpening filter works, and finally each spectral filter will blur the input signal roughly similarly to how a single linear GNN layer blurs the input signal.

In addition, I now realize why the choices made during Lecture 15 affect what can be represented by the model. Sum or average aggregation, whether we use a residual connection, and which non-linear activation function we choose are not simply engineering choices. They directly affect what can and cannot be represented by the network.

This assignment turned GNNs from "black boxes" into open, structured algorithms where I could reason about them using mathematics.
